# Architecture

The project is a **static-file REST API**: Python scrapers fetch HTML from cdep.ro, transform it into structured JSON via Pydantic models, and commit the output to `data/v1/` — which GitHub Pages serves as the API. There is no server.

```
cdep.ro (HTML/PDF) → scrapers/ → schemas/ → data/v1/*.json → GitHub Pages CDN
```

**Critical constraint**: cdep.ro geo-blocks GitHub cloud runners. All production scraping runs on a self-hosted Windows runner in Romania (`.github/workflows/scrape.yml`). Local dev and CI tests run fine everywhere since they use fixtures — they don't hit cdep.ro.

## Layer map

| Directory | Role |
|---|---|
| `scrapers/` | One module per endpoint. Each fetches cdep.ro HTML with `scrapers/_http.py`, parses with `parsel` (CSS/XPath), and returns a list of Pydantic models. |
| `schemas/` | Pydantic v2 models for every resource type. `schemas/common.py` has shared enums/types. |
| `scripts/run_*.py` | CLI entry points for scrapers — load existing data, call scrapers, merge incremental changes, write to `data/v1/`. |
| `scripts/build_*.py` | Derived outputs (comisii aggregated from deputați, amendamente sorted view, feeds, etc.). Run after `run_*`. |
| `data/v1/` | Generated output. **Never edit by hand.** Format: `data/v1/{endpoint}/legislatura-{leg}.json`. |
| `pages/` | HTML pages generated by `generate_html.py`, indexed by Pagefind for full-text search. |
| `tests/` | pytest + syrupy snapshots. Uses HTML fixtures in `tests/fixtures/` — no live network calls. |
| `api/openapi.yaml` | OpenAPI 3.1 spec for all endpoints. |

## Key ID cross-links

- `cdep_idm` — deputy ID; links deputati ↔ voturi ↔ interpelari ↔ motiuni ↔ declaratii
- `cdep_idv` — vote ID; links voturi index ↔ voturi detail ↔ proiecte.vot_final
- `cdep_idp` — legislative project ID; links proiecte ↔ amendamente ↔ ordine-zi ↔ doc-comisii

## Adding a new endpoint

1. Add a Pydantic model in `schemas/`.
2. Add a scraper module in `scrapers/`.
3. Add a `scripts/run_*.py` entry point.
4. Register the output path pattern in `scripts/validate_data.py` and `api/openapi.yaml`.
5. Add snapshot tests in `tests/` with a saved HTML fixture.
